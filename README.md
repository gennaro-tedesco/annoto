This project is an experiment in spec-driven development to determine whether current state-of-the-art of assisted development can produce applications with real usage relevance. The development of this codebase has been almost entirely generated by coding agents, however, with ontinuous review and steering by the human.

## Features

`annoto` is the ultimate chess app. Its core feature is to provide text extraction from hand-written chess scoresheets to `.pgn` files which can subsequently be stored and analysed with an engine.

### File imports

- hand-written text recognition to `.pgn` delegated to the latest cloud models
- import files from `.pgn`, share across various chess apps

### Engine analysis

The app detects any externally available chess engine on device that exposes the OEX protocol for board analysis: the user can choose what engine and what parameters to analyse with. The board view is powered by the flutter lichess libraries to offer the most modern UX experience.

### Lichess integration

`annoto` synchronises with your lichess account to expose all of a user's lichess studies to be comfortably analyed within the app, with the claim that a beautiful and clean UI improves the experience of the current commercial apps.

## Tech stack

This project makes use of [Supabase](https://supabase.com/) as backend-as-a-service to leverage authentication workflows for cloud models: in order to access the hand-written to `.pgn` functionality users must be authenticated within the project.
